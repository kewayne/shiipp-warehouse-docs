# Submit Pre-Alert

Use this endpoint to notify AEORAL that a package is expected to arrive at our warehouse.

Once accepted, the pre-alert is created with a **Pending** status and awaits warehouse processing.

<Info>
  A valid API Key is required for every request.
</Info>

## Endpoint

```http
POST https://api.aeoral.com/v1/prealerts
```

---

## Authentication

Include your API Key in the `X-API-KEY` request header.

```http
X-API-KEY: YOUR_API_KEY
Content-Type: application/json
```

Alternatively, for legacy integrations, the API key may be supplied as a query parameter.

```text
POST /v1/prealerts?api_key=YOUR_API_KEY
```

or

```text
POST /v1/prealerts?key=YOUR_API_KEY
```

> Using the `X-API-KEY` header is strongly recommended.

---

## Request Body

### Required Fields

| Field | Type | Max Length | Description |
| --- | --- | --: | --- |
| UserCode | string | 100 | Customer account or mailbox number. |
| TrackingNumber | string | 255 | Carrier tracking number. |

### Optional Fields

| Field | Type | Description |
| --- | --- | --- |
| Reference | string | Internal reference or order number. |
| Vendor | string | Merchant or sender name. |
| Description | string | Description of package contents. |

### Duplicate Tracking Number

A courier cannot submit the same tracking number more than once.

**HTTP 409**

```json
{
  "status": "error",
  "message": "Pre-alert for this tracking number already exists",
  "data": null
}
```

```json
  }

```

| Vendor | Optional |

---

## Notes

- Only **POST** requests are accepted.
- Each API Key belongs to a single courier.
- Duplicate tracking numbers are prevented per courier.
- Newly created pre-alerts are assigned a status of **Pending**.
- All responses are returned as JSON.
