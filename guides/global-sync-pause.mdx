# Global Sync Pause

Shiipp's **Global Sync Pause** feature allows administrators and managers to temporarily halt the automatic processing and synchronization of packages, manifests, and customer data.

This is incredibly useful when you need to perform system maintenance, hold off on sending updates to courier partners, or temporarily stop automated actions across the platform.

## How It Works

When Global Pause is enabled, the system safely holds all pending tasks and waits. It will stop dispatching new package updates, but it continues to safely store any new packages logged by staff so that you never lose data.

Once you resume synchronization, the system will immediately catch up and process all the packages that were placed on hold during the pause.

## Toggling the Pause State

You can control the sync state directly from the Intake interface.

1. Navigate to the **Scan** page in your dashboard.
2. Look at the top right of the metrics bar, next to the **Sync Delay** indicator.
3. You will see a button indicating the current status (either `Sync Active` or `Sync Paused`).
4. Click the button to toggle the state.

## Role Requirements

To toggle the sync state, your user account must have one of the following roles:

- `admin`
- `manager`
- `warehouse`

If you do not have the required role, you will not be able to change the sync state.

## Team Synchronization

If multiple administrators or managers are using the platform simultaneously, the state of the toggle button is synced across all browsers. If one administrator pauses the sync, the button will automatically update to `Sync Paused` on everyone else's screen within seconds.
