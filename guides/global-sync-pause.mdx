# Global Sync Pause

Shiipp's **Global Sync Pause** feature allows administrators to temporarily halt the background processing of packages, manifests, and customer data. 

This is incredibly useful when you need to perform system maintenance, update credentials for a courier API, or troubleshoot a backend issue without background tasks firing off.

## How It Works

The Shiipp unified worker continuously polls Redis and the database for sync tasks. When Global Pause is enabled, the worker safely skips these heavy operations and idles, while continuing to process mission-critical items like email notifications.

Because the system leverages a centralized Redis key, the pause state propagates **instantly** to all active worker instances.

## Toggling the Pause State

You can control the sync state directly from the Intake interface.

1. Navigate to the **Scan** page.
2. Look at the top right of the metrics bar, next to the **Sync Delay** indicator.
3. You will see a button indicating the current status (either `Sync Active` or `Sync Paused`).
4. Click the button to toggle the state. 

<Info>
  **Note on Data Safety**: When syncing is paused, all packages logged by staff will continue to be securely stored in the database. When you click **Resume**, the worker will immediately begin churning through the queue to catch up. No data is lost during a pause.
</Info>

## Role Requirements

To toggle the sync state, your user account must have one of the following roles:
- `admin`
- `manager`
- `warehouse`

If you do not have the required role, the action will be blocked.

## Multi-Instance Synchronization

If multiple administrators are using the platform simultaneously, the state of the toggle button is synced across all browsers every 15 seconds. If one administrator pauses the sync, the button will automatically update to `Sync Paused` on everyone else's screen.
